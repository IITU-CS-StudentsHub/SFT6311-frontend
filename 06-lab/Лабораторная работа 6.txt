Лабораторная работа 6
Custom Hooks and API Integration. Testing React Applications
Предварительные знания: Introduction to React Ecosystem, Building Components and JSX, State Management with useState, Lifecycle and Effects with useEffect, Advanced State Management with Context API и React Router Basics, Forms and Controlled Components, Performance Optimization
Цель работы
	•	Научиться создавать и использовать кастомные хуки для повторно используемой логики
	•	Подключать внешние API для получения данных и обновления интерфейса
	•	Управлять асинхронными запросами и состоянием загрузки и ошибок
	•	Писать тесты для компонентов и хуков с использованием Jest и React Testing Library
	•	Оптимизировать работу приложения и обеспечивать стабильность при изменениях
	•	Расширять функционал проектов предыдущих лабораторных
Организация групп
Количество студентов: двадцать Размер группы: два или три человека Количество групп: семь
Темы проектов: Группа один и два: Task Tracker Группа три и четыре: Recipe Book Группа пять и шесть: Movie Gallery Группа семь: Mixed Dashboard

Структура 
Часть первая. Custom Hooks
Задача один. Создание кастомного хука useForm Тема: Custom Hooks Условие: Создать кастомный хук useForm, который управляет состоянием формы с несколькими полями. Хук должен возвращать объект состояния формы, функцию для изменения значений полей и функцию для сброса формы. Использовать хук в форме добавления задач, рецептов или фильмов.
Входные данные: ввод текста в поля формы и кнопка Submit Выходные данные: состояние формы обновляется через хук, при Submit вызывается функция добавления элемента в глобальное состояние
Задача два. Создание кастомного хука useFetch Тема: Custom Hooks, API Integration Условие: Создать кастомный хук useFetch, который принимает URL и возвращает данные, состояние загрузки и ошибки. Использовать хук для получения списка задач, рецептов или фильмов с внешнего API или mock API.
Входные данные: URL API Выходные данные: данные загружаются и отображаются на странице, при ошибке выводится сообщение об ошибке, состояние загрузки отображается индикатором
Задача три. Кастомный хук для фильтров useFilter Тема: Custom Hooks Условие: Создать кастомный хук useFilter для фильтрации массива элементов по категории, статусу или тегам. Хук должен принимать массив и параметры фильтрации, возвращать отфильтрованный массив.
Входные данные: массив элементов и параметры фильтра Выходные данные: возвращается массив, соответствующий критериям фильтрации
Задача четыре. Кастомный хук для управления модальными окнами useModal Тема: Custom Hooks Условие: Создать хук useModal, который управляет открытием и закрытием модальных окон. Хук возвращает булево состояние и функции open и close. Использовать его для редактирования элементов, просмотра деталей и подтверждения удаления.
Входные данные: действия пользователя по открытию или закрытию модалки Выходные данные: модальное окно появляется или скрывается
Часть вторая. API Integration
Задача пять. Подключение внешнего API Тема: API Integration Условие: Выбрать внешний публичный API соответствующий проекту (например, API рецептов, фильмов или задач) и интегрировать его в приложение. Отображать полученные данные в карточках с пагинацией.
Входные данные: запрос к API Выходные данные: данные отображаются на странице в формате карточек
Задача шесть. Обработка ошибок и состояния загрузки Тема: API Integration Условие: Реализовать отображение индикатора загрузки при запросе к API и отображение сообщений об ошибке при сбое запроса. Использовать кастомный хук useFetch.
Входные данные: запрос к API Выходные данные: во время загрузки отображается спиннер, при ошибке сообщение об ошибке
Задача семь. Обновление данных через API Тема: API Integration, Context API Условие: Добавить возможность изменения данных через API. Например, добавление новой задачи, редактирование рецепта или фильма. После успешного запроса обновлять глобальное состояние через Context API.
Входные данные: данные формы и кнопка Submit Выходные данные: элемент добавляется или обновляется на странице, глобальное состояние синхронизировано с API
Задача восемь. Удаление данных через API Тема: API Integration Условие: Реализовать удаление элементов через API и обновление глобального состояния после успешного удаления.
Входные данные: выбор элемента и подтверждение удаления Выходные данные: элемент удаляется из интерфейса и из глобального состояния
Часть третья. Testing React Applications
Задача девять. Тестирование кастомного хука useForm Тема: Testing React Applications Условие: Написать тесты для хука useForm с использованием Jest и React Testing Library. Проверить корректность обновления состояния при изменении полей, сброс формы и вызов функции Submit.
Входные данные: имитация ввода данных и вызов функции Submit Выходные данные: состояние формы обновляется согласно действиям
Задача десять. Тестирование компонента формы Тема: Testing React Applications Условие: Написать тесты для компонента формы добавления элемента. Проверить отображение полей, работу ошибок валидации и корректность вызова функций при Submit.
Входные данные: ввод данных и клики по кнопкам Выходные данные: компонент корректно отображает ошибки и вызывает функции
Задача одиннадцать. Тестирование компонента списка Тема: Testing React Applications Условие: Проверить компонент списка задач, рецептов или фильмов. Тесты должны проверять корректное отображение элементов, рендеринг после добавления и удаления.
Входные данные: изменение глобального состояния Выходные данные: интерфейс обновляется согласно изменениям
Задача двенадцать. Тестирование кастомного хука useFetch Тема: Testing React Applications Условие: Проверить работу хука useFetch с mock API. Тесты должны проверять состояние загрузки, получение данных и обработку ошибок.
Входные данные: имитация успешного и ошибочного запроса Выходные данные: состояние загрузки и ошибок соответствует ожидания
